# Introduction to ReactJS #### Dylan Moore
### Intro #### The What * **React** is an open-source (BSD License) library for creating user interfaces on the web * **View** in Model-View-Controller * Simple, Declarative, Composable * Called 'React', but also written as 'ReactJS' or 'React.js'
### Intro #### The Who * Developed by Facebook, Instagram, and [developer community](https://facebook.github.io/react/acknowledgements.html) * Companies using React: Facebook, Instagram, Khan Academy, Netflix, Yahoo, Airbnb, Sony, Atlassian, and others
### Intro #### The Why * Not an MVC Framework - React is a library that encourages reusable UI components * No templates - you use JavaScript instead (no JSTL, etc.) * Unifies markup and view logic - makes views easier to extend * No string concatenation! * Data (i.e. model) updates are super simple * No binding * No manually updates DOM elements
### Intro #### The How * Virtual DOM - React uses `virtual-dom` modules to efficiently change the DOM in place * Developers simply change component state and React/Virtual DOM efficiently take care of the DOM updates * JSX - JavaScript Syntax Extension * Similar to XML (or XHTML), that goes right in your JavaScript * Not processed by the browser, but some type of transform * Mindset * State changes flow down through components, making applications much easier to 'reason about'
